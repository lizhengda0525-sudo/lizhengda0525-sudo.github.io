## 3. 页面与功能

| 页面 | 地址 | 功能 |
|---|---|---|
| 首页 | `/` | 首屏、文章卡片、作者信息、公告、最新文章、标签 |
| 归档 | `/archives` | 按时间查看全部文章 |
| 分类 | `/categories` | 分类统计；点击后筛选文章 |
| 标签 | `/tags` | 标签云；点击后筛选文章 |
| 关于 | `/about` | 显示迁移后的关于页正文 |
| 文章 | `/post/文章标识` | 正文、目录、代码复制、标签、上一篇/下一篇 |

全站支持：

- 点击“搜索”或按 `Ctrl + K`（Windows）/ `Command + K`（macOS）打开全文搜索。
- 搜索范围包含标题、摘要、正文、分类和标签。
- 按 `Esc` 关闭搜索框或手机菜单。
- 点击月亮/太阳按钮切换深浅色；浏览器会记住选择。
- 手机和平板自动切换单栏布局与折叠菜单。

## 4. 本地运行

环境要求：Node.js 22.13 或更高版本、npm 10 或更高版本。

第一次运行：

```powershell
cd D:\KnowledgeBase\MyWeb\linzy-inspired-blog
npm install
npm run dev
```

浏览器访问终端显示的地址，通常为 `http://localhost:3000`。停止预览时按 `Ctrl + C`。

生产构建与自动验收：

```powershell
npm run build
npm test
```

`npm run build` 会先自动读取 Markdown 并更新页面数据；`npm test` 会再次构建，并验证首页、全部功能页、5 篇文章、图片数量、主题、搜索和响应式样式。

`app/content.generated.ts` 是自动生成文件，不要手工编辑。每次启动或构建时，`scripts/generate-content.mjs` 会重新生成它。

## 6. 新增文章

在 `content/posts/` 新建 `.md` 文件。推荐模板：

```markdown
---
title: "文章标题"
date: 2026-07-23T10:00:00+08:00
draft: false
description: "首页显示的文章摘要。"
categories:
  - 分类名称
tags:
  - 标签一
  - 标签二
---

这里开始写正文。

## 二级标题

正文支持列表、链接、图片、表格、引用和代码块。
```

规则：

- 文件名会成为文章网址标识，例如 `my-note.md` 对应 `/post/my-note`。
- `draft: true` 的文章不会发布。
- 日期建议使用带时区的 ISO 格式。
- 分类和标签会自动进入分类页、标签页和搜索索引。
- 未填写摘要时，系统会自动截取正文前 150 个字符。
- 阅读时间按正文字符数自动估算。

保存后，正在运行的开发服务会在重新启动时读取内容；也可手动执行：

```powershell
npm run content:generate
```

## 7. 编辑已有文章

直接编辑 `content/posts/` 中对应文件，然后运行：

```powershell
npm run build
```

不要编辑 `app/content.generated.ts`，因为下次生成时手工修改会被覆盖。

## 8. 使用图片

把图片放入 `public/images/` 的合适子目录，Markdown 中使用从 `/images/` 开始的绝对路径：

```markdown
![图片说明](/images/my-post/example.png)
```

建议：

- 文章图片宽度 1200–1800 像素。
- 照片优先使用压缩后的 WebP/JPEG，界面截图可使用 PNG。
- 文件名避免空格，注意大小写；线上环境会严格区分大小写。
- 首页实际使用 `public/images/site/study-hero.webp`；`study-hero.png` 为迁移保留的原图。替换视觉时同步生成 WebP。

如果要给新文章指定独立首页封面，在 `scripts/generate-content.mjs` 的 `coverMap` 中加入：

```js
"my-note": "/images/my-note/cover.jpg"
```

未指定封面时会使用首页山景图。

## 10. 修改外观

全站样式位于 `app/globals.css`。文件顶部变量控制主要颜色：

```css
--blue: #5c88da;
--blue-deep: #376bbf;
--orange: #ff7b54;
--card: #ffffff;
--text: #4c4948;
--title: #1f2d3d;
```

常用布局：

- `.layout`：页面最大宽度和主布局。
- `.primary` / `aside`：正文与侧栏比例。
- `.post-card`：首页文章卡片。
- `.markdown-body`：文章正文、表格、图片与代码块。
- `html[data-theme="dark"]`：深色模式变量。
- `@media (max-width: 760px)`：手机端样式。

## 11. 搜索机制

搜索完全在浏览器本地运行，不需要数据库。构建脚本会把文章标题、摘要、正文纯文本、分类和标签合并为搜索索引。当前 5 篇文章及未来新增文章都会自动进入搜索。

几十到几百篇文章可继续使用此方式；文章数量很大时再考虑独立搜索服务。

## 13. 常见问题

### 修改 Markdown 后页面没变化

重新启动开发服务，或运行 `npm run content:generate`。正式发布前始终运行 `npm run build`。

### 图片不显示

确认文件确实位于 `public/` 下，Markdown 路径以 `/images/` 开头，并检查文件名大小写。

### 新文章没有出现

检查文件扩展名是否为 `.md`、`draft` 是否为 `false`、头部的 `---` 是否完整闭合，然后运行 `npm run content:generate` 查看结果。

### 搜索不到新内容

执行 `npm run build` 重新生成全文索引，并刷新浏览器。

### 文章地址出现 404

文章网址取自 Markdown 的 `slug` 字段；没有 `slug` 时取文件名。确认访问地址与它完全一致。

### 深色模式颜色不协调

调整 `app/globals.css` 中 `html[data-theme="dark"]` 下的颜色变量，不要只修改浅色变量。
